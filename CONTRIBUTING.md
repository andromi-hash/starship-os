# Contributing to Starship OS

Thank you for your interest in contributing!

## Ways to Contribute
- Bug reports & feature requests (GitHub Issues)
- Code contributions (Pull Requests)
- Documentation improvements
- Manufacturing-specific agent ideas or workflows

## Contribution Process
1. Fork the repo and create a feature branch.
2. Follow the coding standards in `CLAUDE.md` / existing code.
3. Ensure tests pass (`make smoke` or equivalent).
4. Submit a Pull Request with clear description.

## Contributor License Agreement (CLA)
By submitting a contribution, you agree that Absolution Studios may use it under the project’s licenses (MIT for core contributions; commercial license for manufacturing/enterprise extensions). Significant contributions may require signing a formal CLA.

## Code of Conduct
Please follow our (CODE_OF_CONDUCT.md).

## Questions?
Open an issue or contact the maintainers via GitHub or info@sonofabsolution.com.
